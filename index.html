Erlang is a language
a virtual machine
has 4096 children
and is MUNCTIONAL

Created by Joe Armstrong in 1986 for Ericsson

Open sourced in 1998 (GitHub)

Designed for telephone switching

Little or NO downtime

Massive concurrency

OTP - Open Telecom Platform

set of common libraries, utilities, and best practices

ships with erlang

built on the idea of "fail fast"

process lifecycle and supervision

Erlang has...

Functions

{M, F, A} = {module, function, arity}

          -module(math).

          add(X, Y) -> X + Y.
          increment(X) -> add(X, 1).

4 different expression terminators

period, comma, semi-colon,
and nothing!

          foo(X, Y) ->
            Y = math:add(X, Y),
            Z = Y rem 2,
            case Z of
              0 -> true;
              1 -> undefined
            end.

Elementary data types ONLY

no objects

Numbers, Lists, Tuples, Atoms, Funs

atoms

name = value (like Ruby symbols)

generally start with lowercase letter

{tuples}

fixed length ordered list of elements

can contain anything

          X = 4
          Y = 5.5
          Point = {X, Y}
          Location = {point, X, Y}

[lists]

ordered arbitrary length list of elements

singly linked; can contain anything

strings are lists of ascii numbers

          EmptyList = []
          Small = [1, 2, 3]

          AddHead = [0 | Small] 
                  = [0, 1, 2, 3]

          String = "abc"
                 = [97, 98, 99]

Funs

lambdas. closures.

          addX(X) ->
            fun(Y) ->
              X + Y
            end.

          AddTen = addX(10)
          Z = 100
          Result = AddTen(Z) = 110

No loops

only recursion

tail call optimization ;-)

see above

Single assignment

truth statements; synonyms

no mutable state

pattern matching

Simple Examples

          X = 4.
          Y = 2.
          Z = X = Y + 2.

          I = {point, X, Y}.
          {Type, _, _} = I.
          Type = point.

          List = [1, 2, 3]
          [Head | Tail] = List
            % Head = 1
            % Tail = [2, 3]

Recursive Fibonacci in Erlang

          fib(0) -> 0;
          fib(1) -> 1;
          fib(N) ->
            fib(N-1) + fib(N-2).

Crazy bit string pattern matching

one byte TYPE, two byte LENGTH, then payload

          parse(<<Type:8,
                  Length:8,
                  Payload:Length,
                  Rest/binary>>) ->
            % do something
            Rest;
          parse(IncompleteCommand) ->
            IncompleteCommand.

Learn from our experience

Not the solution to every problem

DO Consider if...
high availability is important
you need high levels of concurrency
you are building a core backend system
you like reading man pages with dark humor
DON'T Consider if...
your domain doesn't map well to functional concepts
you don't have time to learn a new paradigm
you need raw power
you want to build a simple CRUD app

Architecture of the system is co-mingled with domain
Strange mix of high and low level abstractions
No mutable state leads to lots of actors.
Lots of actors lead to massive concurrency.
Massive concurrency leads to suffering.
automated testing hasn't been on the scene very long
Our version of TDD new to the community
Latest release has line numbers in stack traces!
Lots of good open source and a good community
