A first taste of
Erlang
What you should know
before
you start your next project
Erlang is a language
and a virtual machine
has 4096 children
and is MUNCTIONAL

Created by Joe Armstrong in 1986 for Ericsson

Open sourced in 1998 (GitHub)

Designed for telephone switching

Little or NO downtime

Massive concurrency

OTP - Open Telecom Platform

set of common libraries, utilities, and best practices

ships with erlang

built on the idea of "fail fast"

process lifecycle and supervision

How does it
do all this?

I don't think that means,
what you think it means

module

flat package of functions

process

a lightweight execution unit

server

a long running process handling messages

application

collection of modules w/ lifecycle

node

an instance of the Erlang VM

Actor Model

Hot Code Upgrades

Node Distribution

Forget Java
this is ERLANG!

Erlang has...

Elementary data types ONLY

no objects

Numbers, Lists, Tuples, Atoms, Funs

4 different expression terminators

period, comma, semi-colon,
and nothing!

No loops

only recurssion

see above

Single assignment

truth statements

no mutable state

pattern matching
