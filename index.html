what

programming language and runtime developed by Ericson

originally propirettary, now opens sourced on github (http://github.com/erlang/otp)

name attributed to both the danish mathematician (Agner Krarup Erlang) and Ericson itself (erricson language)

designed to be the core telephony switching systems that have extreme requirements

little to no downtown

fault tolerant

massively concurrent

multi-paradigm: functional programming through side effects

mix of high level concept and low level details

Open Telecom Platform (OTP) is set of libraries, utilities, and best practices spawned in the trenches

runtime compilation and loading of new code

why

your domain data maps to functional data structures

your domain logic can be broken into many concurrent chunks

you want to leverage multiple computing units in parallel

you hate trying to synchronize your threads

software transactional memory makes your head spin

you are targeting deployment on *nix (Windows if you hate yourself)

your system needs to be fault tolerant

your system has may 9's of uptime requirments

you want to upgrade your system at runtime w/o taking it down

how

nasty syntax

4 different expression terminators

; , .

flat module structure / MFA

functional concepts: higher order functions

actor model

processes / high level than OS processes or threads / lower cost / spin up thousands

NO shared state, all message passing

node / application / process

OTP

gen_server, gen_event, supervisors

macros

release packaging, upgrade releases

node architecture

projects using erlang: riak ....

limited data structures (limiting AND YET powerful)

pid, tuple, list, number, port

lessons learned in the trenches

syntax learning curve takes a bit, especially if not familiar with actor model or functional programming

OTP process architecture is fundamental to how you structure your code and solve your problems

our version of TDD and using test doubles: our changes to Meck
